*추상클래스란?
 - 추상적으로 정의한 후에 객체를 사용할 사용자가 꼭 재정의하여 이용해야 하는 클래스
 - 한 개 이상의 추상메소드를 갖는 클래스
 - 추상클래스는 추상메소드로 인하여 객체 생성이 불가
 - 추상메소드에서는 정의만 하며 구현하지 않음

*추상클래스의 필요성
 - 강제성이 필요할 때
   (상속을 하여 사용 중 자식 클래스에서 강제로 재정의를 해야할 경우)

*final(제한자)의 사용법
 1. 클래스의 앞에 붙는 경우: 해당 클래스 상속 금지
 2. 메소드의 앞에 붙는 경우: 오버라이딩 금지
 3. 변수 앞에 붙는 경우: 상수화되어 변경금지
