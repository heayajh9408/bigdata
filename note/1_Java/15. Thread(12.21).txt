*Thread란?
 - 하나의 프로그램이 동시에 여러 개의 일을 수행할 수 있게 해주는 것을 말한다.
 - n개의 스레드가 객체 하나를 공유, n개의 스레드가 객체 n개를 사용하는 방식이 있다
 - 생성 - > 실행상태<->실행가능상태 -> 완료 or 대기

#장점
 - 효율적인 자원의 사용 , 응답성 향상, 작업이 분리되어 코드가 간결해진다
#단점
 - 동기화에 주의해야 하고 프로그래밍할 때 고려해야 할 사항이 많다

*Thread의 문법
 - Runnable 인터페이스 구현을 통한 Target과 Thread
 - Thread 클래스 상속을 통한 Thread
 - Runnable 인터페이스 구현을 통한 Target과 Thread (target공유)
 - 객체1개, 스레드n개
 - 객체n개, 스레드n개

# Synchronized(동기화)
 - 먼저 수행되는 스레드의 모든 작업이 끝날 때까지 다른 스레드는 기다리도록 하는 예약어
