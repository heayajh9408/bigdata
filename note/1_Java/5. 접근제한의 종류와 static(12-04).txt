12.04 접근제한의 종류와 static

# 멤버변수 선언

①접근제한자 ②static/final ③ 데이터형 ④변수명

1. 접근제한자
- 접근제한이란 클래스의 데이터나 메소드에 대해서 다른 클래스로부터 접근을 제한하는 것이다.
- 접근제한자의 종류: public > protected > default > private
  *public : 접근을 제한하지 않는다 ( 다른 모든 클래스에서 사용 가능)
  *protected : 해당 클래스와 동일한 패키지 or 상속받은 클래스일 경우에만 사용가능
  *default : 접근제어자가 명시되지 않을 경우의 디폴트 접근제어자( 같은 패키지내의 클래스는 public 권한을 갖고 접근 가능)
  *private : 해당 클래스만이 사용 가능(외부에서는 절대로 접근 불가) 

2. 멤버변수의 초기화 시기와 순서
- 클래스 변수의 초기화 시점 : 클래스가 처음 로딩될 때 단 한번
- 인스턴스 변수의 초기화 시점 : 인스턴스(객체)가 생성될 때 마다

3. static : 클래스변수
- 클래스 변수는 객체들의 수와 상관없이 하나만 생성된다
- 한 클래스로부터 생성된 모든 객체들은 클래스 변수를 공유
- 객체들끼리 공유하고 싶은 정보가 있을 때 사용하는 것으로 일명 "깍두기"

**final(종단변수)은 원주율과 같은 빈번이 사용되는 상수를 정의할 때 사용**
