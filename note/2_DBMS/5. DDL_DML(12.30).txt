* DDL, DCL, DML

*DDL(Data Definition Language): 데이터와 그 구조를 정의
 O CREATE : 데이터 베이스 객체 생성: CREATE TABLE 테이블명
  - 데이터 형의 종류들 : VARCHAR2(SIZE), NUMBER, DATE, LOB 등
 O ALTER: 기존에 존재하는 데이터베이스 객체를 다시 정의
  - ADD를 통해 새로운 컬럼 추가
  - MODIFY를 통해 컬럼의 데이터 타입이나 크기 등을 변경
  -S ET UNUSED를 통해 컬럼을 사용을 논리적으로 제한, 삭제가 되는 것은 아님
 ODROP: 데이터 베이스 객체를 삭제 
 OTRUNCATE: 데이터베이스 객체 내용 삭제
  - 기존에 사용하던 테이블의 모든 로우를 제거

#데이터 딕셔너리
 O 사용자가 테이블을 생성하거나 사용자 변경 등의 작업 시 자동으로 갱신되는 테이블
 O 사용자는 딕셔너리의 내용을 직접 수정 또는 삭제가 불가능하다
#데이터 딕셔버리뷰
 O 자료 조회가 불가능하기에 데이터 딕셔너리에서 파생되어 제공되어지는 뷰
 O 종류들
  - DBA_XXX : 데이터베이스 관리자만 접근 가능한 객체 등의 정보 조회
  - ALL_XXX  : 자신 계정 소유 또는 권한을 부여 받은 객체 등에 관한 정보 조회
  - USER_XXX: 자신의 계정이 소유한 객체 등의 관한 정보 조회 

2. DML(Data Manipulation Language): 데이터 검색, 수정 등
 O INSERT: 데이터베이스 객체에 데이터를 입력
  - INSERT INTO 테이블명 VALUES 값
  - NULL 값을 삽입할 경우 명시적인 방법과 암식적인 방법이 있음
  - 서브쿼리로 데이터 삽입 가능
 O UPDATE: 기존에 존재하는 데이터베이스 객체 안의 데이터수정
  - WHERE 절을 사용하지 않을 경우 테이블에 있는 모든 행이 수정됨으로 주의할 것
 O DELETE: 데이터베이스 객체에 데이터를 삭제
 O SELECT: 데이터베이스 객체로부터 데이터를 검색

# 테이블의 제약조건들
 - DBMS에 부적합한 데이터가 테이블에 삽입되는 것을 방지하기 위해 CONSTRAINT 사용
 - NOT NULL : 이 열은 NULL 값을 포함하지 않음
 - UNIQUE: 테이블의 모든 행에 대해 유일해야 하는 값을 가진 열을 지정
 - PRIMARY KEY: 유일하게 테이블의 각 행을 식별
 - FOREIGN KEY: 열과 참조된 테이블의 열 사이의 외래키 관계를 적용하고 설정, REFERENCES 예약어 사용
 - CHECK: 참이어야 하는 조건을 지정
 - DEFAULT: 제약조건은 아니만 사용할 수 있음
